AI Research Paper Reviewer Documentation

Technical documentation bundle for the FastAPI, LangGraph, Gemini, Semantic Scholar, and React-based research paper review platform.

1. System Overview

AI Research Paper Reviewer accepts a research paper by direct PDF upload or arXiv URL, extracts paper content, enriches it with related literature from Semantic Scholar, evaluates it through a two-node LangGraph workflow backed by Google Gemini, stores the structured result in SQLite, and renders the final report in a React UI.

  • FastAPI backend for orchestration, persistence, progress streaming, and PDF generation
  • React frontend for upload, history browsing, live progress, and report display
  • LangGraph pipeline with exactly two nodes: research and review
  • Persistent raw Gemini outputs for debugging and auditability

2. Main Runtime Flow

User -> Frontend UploadZone -> POST /api/review -> FastAPI start_review
-> create Review + ProgressEvent -> run_pipeline(...) in background
-> LangGraph review_graph
-> research_node
   -> extract_paper(...)
   -> search_related_papers(...)
   -> Gemini Flash
-> review_node
   -> Gemini Pro
-> persist results + PDF generation
-> SSE progress stream completes
-> Frontend fetches GET /api/review/{id}
-> User sees report and raw Gemini outputs

7. Database Schema Summary

  • reviews: top-level review metadata and outputs
  • review_dimension_scores: six-dimension scoring details
  • related_papers: Semantic Scholar paper list
  • progress_events: streamed progress messages

8. Operational Notes

  • Use Python 3.11+
  • Run alembic upgrade head before starting the backend
  • WeasyPrint may need native OS libraries for real PDF rendering
  • Older SQLite databases are patched for compatibility at backend startup

9. Troubleshooting Summary

  • Missing Gemini key: check root .env
  • Missing debug columns: run migrations or restart backend for compatibility patch
  • Stuck processing: inspect backend traceback and raw Gemini outputs on the review page
  • Frontend cannot connect: confirm backend on localhost:8000
